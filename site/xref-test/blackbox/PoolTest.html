1046    *
1047    * - You (or your thread) do a successful claim, and triumphantly stores it
1048    *   in the ThreadLocal cache.
1049    * - You then return the object after use, so now it's back in the
1050    *   live-queue for others to grab.
1051    * - Someone else tries to claim the object, but decides that it has expired,
1052    *   and sends it off through the dead-queue to be reallocated.
1053    * - The reallocation fails for some reason, and the slot is now poisoned.
1054    * - You want to claim an object again, and start by looking in the
1055    *   ThreadLocal cache.

1058    *   *not* pull it off of the live-queue â you cannot just put it on the
1059    *   dead-queue, because that could lead to unbounded memory use.
1060    * - Instead, it has to be marked as live, and we instead have to wait for
1061    *   someone to pull it off of the live-queue, check the poison again, and
1062    *   *then* put it on the dead-queue.

1064    *   wrapped in a PoolException.

1066    *   so if you quickly catch the exception and try to claim again, you will
1067    *   find the same exact poisoned slot and go through the same routine, that
